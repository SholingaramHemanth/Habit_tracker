# Technical Manifesto

## 1. Introduction
Aura is built on the principle that discipline should be as engaging as a high-budget video game. This project implements advanced state management and real-time visualization to bridge the gap between intent and action.

## 2. Architecture
The app follows a strict modular pattern, separating UI components from the core logic in Dashboard.tsx.

## 3. The 12 Settings Categories
### General
1. Profile: Manage avatar and personal declarations.
2. Theme: Switch between Void (Dark) and Ethereal (Light).
3. Language: Localized aura support.
4. Week Start: Global synchronization.
5. Goals: Define your trajectory.

### Notifications
6. Reminders: Precision timing.
7. Smart Alerts: AI-driven behavior correction.
8. Snooze Protocol: Flexible time-blocking.
9. Streak Alerts: Celebration of consistency.

### Privacy & Data
10. Data Lock: Biometric and PIN protection.
11. Export: Full CSV/PDF data sovereignty.
12. Sync: Cross-cloud synchronization.

## 4. Notification Logic
Our notification engine uses a dual-layer check:
- Layer 1: Time-based triggers.
- Layer 2: Behavior-based triggers (Smart Reminders).

Every notification includes "Mark as Done" and "Snooze" actions to reduce friction.
